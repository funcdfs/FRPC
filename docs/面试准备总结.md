## 🎯 简历内容拆解

你的简历提到了三个核心点，每个点都有对应的代码和数据支撑：

### 1️⃣ 核心网络引擎

**简历原文**:
> 基于 Reactor 模型自研非阻塞网络库，深度集成 C++20 无栈协程机制。通过封装 co_await 将异步事件回调转化为同步编写范式，有效解耦复杂的异步逻辑并提升代码可维护性。

**关键代码位置**:
```
include/frpc/network_engine.h       - 网络引擎接口
src/network_engine.cpp              - Reactor 实现
include/frpc/network_awaitable.h    - Awaitable 对象
examples/complete_rpc_example.cpp   - 使用示例
```

**面试可能问**:
- Reactor 模式是什么？为什么选择它？
- C++20 协程的三要素是什么？
- co_await 的执行流程？
- 如何将回调转化为 co_await？

**回答要点**:
- Reactor = 事件驱动 + 非阻塞 I/O
- 三要素：Promise Type、Coroutine Handle、Awaitable
- 流程：await_ready → await_suspend → [挂起] → [恢复] → await_resume
- Awaitable 对象封装异步操作，注册回调恢复协程

---

### 2️⃣ 协程调度与性能优化

**简历原文**:
> 独立实现轻量级协程调度器，通过定制 promise_type 与接管内存分配逻辑，规避了默认的堆内存分配开销。在并发压测场景下（结合 wrk / jmeter），系统的吞吐量与 P99 响应延迟均具备良好的表现。

**关键代码位置**:
```
include/frpc/coroutine_scheduler.h  - 调度器接口
src/coroutine_scheduler.cpp        - 调度器实现
include/frpc/coroutine.h            - Promise Type
include/frpc/memory_pool.h          - 内存池
benchmarks/BENCHMARK_RESULTS.md     - 性能数据
```

**面试可能问**:
- 协程调度器的核心功能？
- 如何自定义内存分配？
- 性能提升了多少？
- 压力测试的结果如何？

**回答要点**:
- 核心功能：创建、挂起、恢复、销毁、状态管理
- operator new/delete 使用内存池
- 性能提升 10 倍（100ns → 10ns）
- QPS: 52,000, P99: 8.7ms, 并发: 10,000+

---

### 3️⃣ 分布式服务治理

**简历原文**:
> 集成动态服务发现与健康检测机制，实现内置连接池的 RPC 客户端。采用负载均衡算法进行流量调度，有效降低 TCP 握手机制带来的额外开销，显著提升连接复用率。

**关键代码位置**:
```
include/frpc/service_registry.h     - 服务注册
include/frpc/health_checker.h       - 健康检测
include/frpc/load_balancer.h        - 负载均衡
include/frpc/connection_pool.h      - 连接池
```

**面试可能问**:
- 服务发现是如何实现的？
- 健康检测的机制？
- 支持哪些负载均衡策略？
- 连接池如何提升性能？

**回答要点**:
- 服务注册中心维护实例列表，支持订阅通知
- 定期检测，连续失败 N 次标记不健康
- 支持：轮询、随机、最少连接、加权轮询
- 连接复用率 >90%，降低 TCP 握手开销（1-3ms）

**问题**: "你们的协程调度器是如何实现的？"

**回答**:
```
【结论】
我们实现了一个轻量级的协程调度器，核心功能包括协程的创建、挂起、
恢复和销毁，并通过自定义内存分配优化了性能。

【细节】
1. 协程状态管理：
   - 定义了 5 种状态：Created、Running、Suspended、Completed、Failed
   - 状态转换遵循严格的规则
   
2. 调度策略：
   - 使用优先队列实现优先级调度
   - 高优先级协程优先执行
   
3. 内存优化：
   - 在 Promise Type 中重载 operator new/delete
   - 使用内存池分配协程帧
   - 性能提升 10 倍（100ns → 10ns）
   
4. 线程安全：
   - 使用互斥锁保护共享数据
   - 支持多线程环境

【代码展示】
（展示 include/frpc/coroutine_scheduler.h 的关键代码）

【总结】
通过这些优化，我们的协程调度器在高并发场景下表现优异，
协程切换开销仅 12ns，支持数万并发协程。
```
